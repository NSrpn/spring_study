Урок 2:
1. Изменена система авторизации (теперь это происходит через Spring Security)
1.1. У пользователей добавились роли: USER и ADMIN
1.2. При регистрации с главной страницы создается пользователь с ролью USER
1.3. Администратор может добавлять книги, создавать новых пользователей (в том числе администраторов).
1.3.1. Администратором счтается пользователь с соответствующим флагом, либо с логином admin.
1.3.2. Администратор может поставить флаг принудительной смены пароля при следующем входе в систему.
2. Добавлено сохранение и загрузка файлов.
   Возможно 2 вида хранилища: БД и Файловая система.
   Настройка храилища производится в файле: resources/fileStorage.properties.
2.1. БД (fileStorage.type=DATABASE)
     Файлы хранятся в таблице Files в Base64.
2.2. Файловая система (fileStorage.type=FILE)
     Файлы хранятся по пути, определенном в параметре:
        fileStorage.folder=E:\\projects\\nsrpn\\files
3. Перекрыта страница 404 и реализован обработчик ошибок.
4. Сборка и отладка переведена на tomcat, ибо у jetty проблемы со смешанной инициализацией (web.xml + WebApplicationInitializer)
